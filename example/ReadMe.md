# Example
This example demonstrates a full round-trip: modification by user in UI -> config update -> response to update by data controller -> UI update

## Usage
Make sure that you run the example from the ConfigProvider root directory.
Adjustments made by the example application will be saved to the file build/audio.demo.json
While the application is running, try to edit the file, saved changes will reflect in the UI. Have fun playing around with it and report bugs if you find any.
